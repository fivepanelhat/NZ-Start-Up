# Privacy Act 2020 — Product Checklist

## Default product posture

| Choice | Rationale |
|--------|-----------|
| Local company memory | Minimise third-party hosting of financial/legal data |
| No secrets in git | Security + IPP storage principles |
| Draft vs submitted separation | Accuracy and purpose limitation |
| Audit log | Accountability |

## Before collecting personal information

- [ ] Purpose stated in plain language  
- [ ] Only fields needed for that purpose  
- [ ] Retention period defined  
- [ ] Access/correction path documented  

## Agent-specific

- Do not scrape personal data into marketing lists without a lawful basis  
- CRM hygiene ≠ silent enrichment from opaque sources without disclosure  
- When using cloud LLMs, minimise PII in prompts; prefer redaction  

## Breach readiness

If a future hosted version holds personal information, maintain incident response for notifiable privacy breaches (OPC guidance).
