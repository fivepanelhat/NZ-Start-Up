# CAT Gold / Diamond / Platinum — Applied to NZ Start-Up in a Box

Operational summary of Coastal Alpine Tech architectural standards for this product. Full skill: `skills/cat-architectural-standards`.

## Gold — Workflow-native design

**Process-first.** The fleet is a digital reflection of the NZ founder lifecycle:

1. Formation → 2. Compliance cadence → 3. Funding/RDTI → 4. Market validation →  
5. GTM → 6. Content → 7. Finance → 8. Capital raise prep → 9. Legal drafts → 10. Board rhythm

**Characteristics in this repo:**

- Unbroken data chain via **company memory** (one source of truth)
- Lifecycle modularity = one skill per lifecycle phase
- Deep NZ industry integration (Companies Office, IRD, RDTI, EDAs)
- User-centric automation (drafts, checklists, reminders — not magic filing)

## Diamond — Enterprise-grade foundation

**Tier-1-in-a-box technical baseline** for the pack itself:

| Concern | Implementation |
|---------|----------------|
| Stack honesty | Skills + optional Aether; desktop later; SaaS last |
| CI | GitHub Actions skill validation + pytest |
| Security | Guardrails, no secrets, HITL matrix |
| Compliance readiness | Privacy Act, legal ceilings documented |
| Observability | Audit log schema + weekly board report |
| Packaging | Versioned skills, CHANGELOGs, proprietary licence |

Diamond does **not** mean overbuilding Kubernetes for a solo founder. It means production hygiene appropriate to the delivery shape (skills pack first).

## Platinum — Intelligent self-improving system

Gold workflow + Diamond reliability + **AI engine / flywheel**:

1. **Strategic data capture** — RDTI logs, decisions, pipeline outcomes, content performance  
2. **Centralised AI engine** — Board orchestrator + specialists (Aether pattern)  
3. **Continuous improvement** — better drafts from retained company memory  
4. **Predictive value** — runway alerts, deadline risk, grant fit scoring  

**Platinum Edge (portfolio alignment):** local Ollama / edge inference for low-stakes tasks; frontier API only when founder opts in for high-stakes drafting.

## Decision protocol (every task)

1. Classify primary tier (Gold / Diamond / Platinum)  
2. State classification  
3. Apply tier rules + HITL  
4. Record material decisions in company memory  

## Anti-patterns

- Platinum without Gold workflow (“random agents”)  
- Claiming Diamond without CI, security, or audit  
- Gold linear checklists that never write memory (blocks flywheel)  
- Skipping cultural sovereignty on any tier
